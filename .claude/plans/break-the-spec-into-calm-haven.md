# 计划：将 SEPC.md 规格拆分为可执行里程碑

## Context

SEPC.md 是一份完整的基金竞品分析工具规格文档，包含4个MVP阶段（Phase 1-4），但每个阶段的粒度太粗，无法直接作为可执行的开发任务。用户要求将其拆分为**可运行、可演示、可验收**的里程碑，并增加 Phase 0 前置环境准备。

## 最终方案

已保存设计文档到 `docs/superpowers/specs/2026-04-28-milestone-breakdown.md`。

拆分为 **5个Phase，共18个里程碑**：

| Phase | 里程碑 | 内容 |
|-------|--------|------|
| Phase 0 | M0.1-M0.9 | 环境准备（.env、目录脚手架、虚拟环境、API验证脚本、配置管理、测试基础设施、Playwright安装） |
| Phase 1 | M1-M3 | 数据基础设施（数据库+数据获取、增量更新、数据质量校验） |
| Phase 2 | M4-M6 | 核心分析（业绩指标、竞品推荐、持仓对比） |
| Phase 3 | M7-M8 | 报告输出（HTML报告、PDF导出） |
| Phase 4 | M9-M10 | UI优化（三栏布局、报告集成） |

## 关键文件

- `docs/superpowers/specs/SEPC.md` — 原始规格文档
- `docs/superpowers/specs/2026-04-28-milestone-breakdown.md` — 里程碑拆分设计文档

## Phase 0 筛选过程

根据"如果不做这个动作，后续阶段还能进行么？会重复遇到同一个卡点么？"原则筛选：

| 候选项 | 决策 | 理由 |
|--------|------|------|
| API连通性验证 | ✅ 纳入 | 认证问题会阻塞全部后续 |
| 配置管理模块 | ✅ 纳入 | 多处硬编码会反复改 |
| 测试基础设施 | ✅ 纳入 | 没有mock fixture无法写测试 |
| Playwright浏览器安装 | ✅ 纳入 | PDF导出硬依赖 |
| 类型检查/代码规范 | ⏭ 排除 | 不阻塞开发，可后续补 |
| 日志框架 | ⏭ 排除 | print够用，可后续统一 |
| 错误处理基础设施 | ⏭ 排除 | 各模块编写时逐步建立即可 |

## 验证

- 已确认每个里程碑有明确的入口命令
- 已确认每个里程碑有可演示的功能
- 已确认每个里程碑有客观的验收标准
- 已梳理里程碑之间的依赖关系
